6. Which system integration tools can AboveE BPM Engine Design?

Answer:

(1) In terms of forms, a JSP webpage can directly call API and is integrated with the process engine. PHP/ASP/ASP .NET can integrate with the process engine though web services.

(2) Process and integration with other systems to communicate with them can be done by using the Adapter of the AboveE Integration Manager.

7. What process design tools are provided by AboveE (Web/Client)?

Answer:

Web-based form information design and page layout tools (webForm) are provided. Client-end tools include process designers and forms approval module design tools (SignFlow).

8. Does Power Process Suite support process information version control features?

Answer:

Yes, when a new process version is registered, new processes will be conducted using the new process version. Existing processes will continue to be conducted using the existing process version.

Yes, Adaptor modules can be designed by the user.

10. Does Power Process Suite support parent-child process design (modulation of process design)?

Answer:

Yes, Power Process Suite process child process modules that can call pre-designed processes.

11. Does Power Process Suite include a complete process control API?

Answer:

Yes, Power Process Suite provides processes and forms API that includes start, stop, approval, pause, continue, get to be processed items, and etc…. It can also integrate with foreign systems.

----------------------------------------------------------------------------------------------------------------------------------

12. How can the API by AboveE be used by Web Services and non-related AP?

Answer:

The API in the Power Process Suite provides a web service interface. Other applications can communicate with the process engine through this interface. As well, AboveE also provides a web service adapter to allow processes to call other web service systems.

13. Can the format of the alert emails, such as flag, title, attachment, and message body be customized?

Answer:

Yes, the alert emails support HTML formatting emails. Users can customize the title, message body, and language settings. However the message flag and attachments are not supported.

14. Does Power Process Suite support alert settings? What are the configuration options?

Answer:

Yes, approval alerts can be consists of pre-approval alerts and post-approval alerts. Post-approval alerts can be configured to multiple notices, and can dynamically adjust the frequency of the alerts, receiver, and contents of the mail. Post-approval alerts can send different emails according to the current status (normal, past-due, exception).

22. What languages are supported in the Power Process system interface?

Answer:

The client end tools would not automatically detect the system language setting of the OS. Currently there are only English, Simplified Chinese, and Traditional Chinese webpage tools. Resource bundling is supported and can automatically adjust the displayed language based on the Internet Explorer environment. Gut this is only supported in Power webForm. It is not supported in user designed forms or the data entered in forms,

23. How to calculate the upgrading cost and method from Power Process System (Professional Edition) to Power Process System (Enterprise Edition)?

Answer:

The version of Power Process Suite needs to be installed on weblogic/websphere with the Enterprise file version. Organizational structure and processes can be exported and imported from the Professional version to the Enterprise version. There is not need to redesign them. No adjustments need to be made with the database. WebPages and files need to be copied over. Only the price difference between the two versions needs to be paid.

----------------------------------------------------------------------------------------------------------------------------------

24. Does Power Process Suite support JDK1.5?

Answer:

It currently supports only JDK 1.4.2 a nd does not support the 1.5 version. However, it is currently being tested under the 1.5. Any new version would introduce new problems that will be quickly fixed in the next version. It currently runs very stable on version 1.4.2. But we are not sure whether there would be a stable version of JDK 1.5. It is currently in testing, if it is stable, we would support it in later versions.

----------------------------------------------------------------------------------------------------------------------------------

26. Which version of Apache does Power Process Suite support?

Answer:

Because Apache HTTP Server is not a Java Container. But the webForms system was developed with JSP technology. Therefore, it cannot and there is not need to use the Apache HTTP server. AboveE Power Process BPM system already includes an Apache Jakarta Tomcat server to be used as the web/JSPserver.
