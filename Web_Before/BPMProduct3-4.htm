BPM Solution
 
 
 

Faced with the rapid development in B2B e-commerce, businesses and other customers are becoming increasingly connected to suppliers to exchange data though the Internet, resulting in the increasing need to integrate systems. AboveE provides the RosettaNet Channel Manager (RNCM) solution to assist businesses to quickly join the global industry supply chain.

RNCM is a B2B e-commerce business management platform that is compliant with the RosettaNet transaction standard. It can be easily configured to execute the information in the Trading Partner Agreement as required by RosettaNet. Also, it has a complete query and tracking feature to provide records of process execution status, PIP (Partners Interface Process) from exchanges, contents of exchanged information, and error tracking. The user interface is web-based, with easy to use operation methods. It has a built-in fast Level 1 and Level 2 testing. These features reduce the time system administrators needed to learn and maintain the system. When the system is being deployed or after it goes online, if there are problems with a trade partner, either the system administrator or a B2B internal user can track and quickly respond to them by using a web browser.

Simple System Integration Methods

RNCM not only provides the ability to exchange data with transaction partners, but can also provide data from transaction partners directly to backend corporate systems using the following three methods:

 
 

1• Through the Power Process BPM process integration feature, the developed process name can be configured onto RNCM so the data from transaction partners can be directly connected to internal processes and onto the backend system, resulting in real time date from transaction partners around the world.

2• RNCM has an internal Adapter Manager that manages all customized adapters and backend system connections. Through integration using the adapter, the internal requirements of businesses can be customized and the ability to integrate with backend systems is thus platform independent.

3• RNCM has as internal Adapter can be configured by using a wizard to transfer data received instantaneously to the appropriate database to assist with the rapid development of backend systems.

 
Transaction Agreement Allocation

As different PIPs are developed, RNCM provides a graphical user interface (GUI) to system administrators to manage the data flow; then the appropriate transaction method is customized based on the PIP (Partners Interface Process) agreement. After configuring the agreement, Level 1 and Level 2 test can be conducted with transaction partners. The results of these tests would be beneficial to the development of an integrated backend.

 
Simple Form Query and Monitoring

Though querying and monitoring RNCM forms, the administrator or user can directly query the transaction status using the form number. For example, the 3A 4 Purchase Order Request has a Document ID, which is the form number of the purchasing form. If there is need to resend a transaction message, a resend feature in the query results can be used to resend the message. RNCM also provides users with the mechanisms to upload data format files (XSL/XSLT) to allow queries of transaction information to be presented in the predefined format.

 

Standard Message Generator Wizard

RNCM allows administrators to generate standard RosettaNet Messages without any programming. Standard transaction messages received from transaction partners can be exported using the same wizard. This feature can accelerate the deployment speed and lower costs associated with human resources and time.

 
History Viewer

RNCM provides administrators with Message Log and System Log to investigate problematic transactions using query and viewing options. Administrators can use either the DEBUG, INFO, WARN, ERROR, or FATAL level to view the transaction messages to quickly find problems.
